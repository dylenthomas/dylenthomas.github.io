  <p>This is the first post for this project I am making on this website. So, this is just an overview of the current system.</p>

  <h3>Compute Node</h3>
  <p>The compute node (my desktop server) processes all the live audio and is checking for the keywords I have defined.</p>

  <p>It collects audio with two modified Blue Snowballs. 
    The modifications are just removing the internals from the housing and placing them in 3D printed mounts I have on my walls.</p>

  <p>The microphones are shown here:</p>
  <img src="/assets/images/lars/mic1.png" alt="Microphone 1">
  <img src="/assets/images/lars/mic2.png" alt="Microphone 2">

  <p>They are connected through USB 3.0 extension cables. 
  Interestingly the first microphone was so far from my desktop I had to buy a powered extension to boost the signal, or else it wouldn't register.</p>

  <p>To prevent the ASR model (currently <a href="https://github.com/SYSTRAN/faster-whisper">faster-whisper</a>) 
  from running on audio that contains no speech and wasting resources, I have included a Voice Activity Detection (VAD) model to only run ASR on audio which contains speech. 
  The model I am using is <a href="https://github.com/snakers4/silero-vad">silero-vad</a>.</p>

  <p>Then once a keyword is identified it tells the Raspberry Pi I have connected to my hardware what keyword it found.</p>

  <h3>Edge Device</h3>
  <p>The edge device connected to my hardware that executes commands is a Raspberry Pi-4. Shown here:</p>
  <img src="/assets/images/lars/raspberrypi_housing.png" alt="Raspberry Pi 4 Housing">

  <p>It is mounted to my wall in a custom housing and is wired to a relay board (the blue board). 
  From there, by just switching a relay I can turn my lights on/off, or perform some other action I have yet to think of.</p>

  <p>It also has two wires connected to IR diodes that are housed within my Kenwood Amp. 
  With that I can emulate the IR codes the remote makes, and with the Raspberry Pi I can control my amp as if it was the remote.</p>

  <p>All of these actions are executed when a specific keyword is received from the compute node. 
  When the keyword is received, the Raspberry Pi determines which action the keyword represents and executes it.</p>

  <h3>Currently</h3>
  <p>I am currently refactoring the code base to switch to a C-first approach.</p>

  <p>Currently the compute node’s main loop is in Python so that it can easily run an ASR model. 
  Then I would use a shared object library I made in C to interact with the microphones. 
  But this felt clunky and annoying, so I am switching the main loop to C. Then I will use Python to only interact with the ASR model.</p>

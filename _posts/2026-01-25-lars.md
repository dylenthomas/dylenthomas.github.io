This is the first post for this project I am making on this website. So, this is just an overview of the current system.

### Compute Node
The compute node (my desktop server) processes all the live audio and is checking for the keywords I have defined.

It collects audio with two modified Blue Snowballs. 
The modifications are just removing the internals from the housing and placing them in 3D printed mounts I have on my walls.

The microhpones are shown here:
![Microphone 1](assets/images/lars/mic1.png)
![Microphone 2](assets/images/lars/mic2.png)

They are connected through usb 3.0 extensions cables. 
Interestingly the first microhpone was so far from my desktop I had to buy a powered extension to boost the signal, or else it wouldn't register.

To prevent the ASR model (currently [faster-whisper](https://github.com/SYSTRAN/faster-whisper) to get accurate real-time transcription), from running on audio that contains no speech, and wasting resources.
I have included a Voice Activity Detection (VAD) model to only run ASR on audio which contains speech, the model I am using is [silero-vad](https://github.com/snakers4/silero-vad).

Then once a keyword is identified it tells the Raspberry Pi I have connected to my hardware what keyword it found.

### Edge Device
The edge device connected to my hardware that executes commands is a Raspberry Pi-4. Shown here:
![Raspberry Pi 4 Housing](assets/images/lars/raspberrypi_housing.png)

It is mounted to my wall in a custom housing and is wired to a relay board (the blue board).
From there which just switching a relay I can turn my lights on/off, or some other action I have yet to think of.

It also has two wires connected to IR diodes that are housed within my Kenwood Amp.
With that I can emmulate the IR codes the remote makes, and with the Raspberry Pi I can control my amp as if it was the remote.

All of these actions are executed when a specific keyword is recieved from the compute node. 
When the keyword is recieved the Raspberry Pi determines which action the keyword represents and executes it.

### Currently
I am currently refactoring the code base to switch to a C first approach. 

Currently the compute nodes main loop is in Python so that it can easily run a ASR model.
Then I would use a shared object library I made in C to interact with the microphones. But this felt clunky and annoying, so I am switching the main loop to C.
Then I will use Python to only interact with the ASR model.
